# Split — онбординг‑анимация (3D, реагирует на наклон/мышь)

Самодостаточная статическая страница: 3D‑сцена онбординга, которая реагирует на
**наклон телефона** (гироскоп) на мобильных и на **движение мыши** на десктопе.
Готова к деплою на **GitHub Pages** без сервера и без сборки.

## Что это технически

**Реальные 3D‑меши, рендеринг через [three.js](https://threejs.org) (r160), WebGL.**
Это НЕ слоистый 2.5D на CSS‑трансформах. Все объекты — настоящая геометрия:

- рамка телефона — `ExtrudeGeometry` (скруглённый контур) + эмиссивная зелёная кромка;
- знак **%** — два экструдированных кольца и перекладина (`ExtrudeGeometry`);
- спираль — `TorusGeometry` с вершинными цветами (teal→green);
- монеты ₽ — `CylinderGeometry` + рельефный `₽` через displacement/bump‑карту;
- галочка ✓ — `CylinderGeometry` с рельефной `✓`.

Все текстуры (золото, рельеф глифов, environment‑карта отражений) генерируются
процедурно на `<canvas>` в рантайме — **внешних картинок‑ассетов нет вообще**.
Наклон/мышь вращают всю 3D‑группу (`world`), перспективная камера даёт честный
параллакс‑по‑глубине между передними (монеты, %) и задними слоями.

**Интерфейсные элементы онбординга** (по макету Figma) добавлены поверх сцены
как обычный HTML/CSS‑оверлей, не затрагивая 3D и фон:

- заголовок «Повышенный кэшбек для клиентов Сбера» — шрифт **Geologica SemiBold**,
  30px, белый, по центру, вверху;
- кнопка **«Супер»** — белая, радиус 8px, текст **Inter Bold** 16px `#121212`,
  закреплена снизу.

Оверлей лежит выше зерна/виньетки (кнопка остаётся чистой и кликабельной) и
выровнен в центрированную «телефонную» колонку 412px, как во фрейме Figma.

> Исходник — компонент из дизайн‑бандла «Анимация онбординга продукта», который
> был завязан на проприетарный рантайм (`DCLogic` / `support.js`, React‑подобные
> рефы и `<x-dc>`). Здесь он перенесён в чистый фреймворк‑независимый `index.html`.

## Структура

```
.
├── index.html                 # вся страница: разметка, стили, сцена и логика
├── libs/
│   └── three.module.min.js    # three.js r160, вендорится локально (≈655 КБ)
├── .nojekyll                  # чтобы GitHub Pages не трогал файлы
├── .gitignore                 # .claude/ и _site/ не попадают в репозиторий
├── .github/workflows/deploy.yml  # авто‑деплой на Pages (опционально)
└── README.md
```

> Деплоятся только `index.html`, `libs/` и `.nojekyll`. Локальный конфиг
> предпросмотра `.claude/` исключён через `.gitignore` и не публикуется.

Все пути к ассетам **относительные** (`./libs/three.module.min.js`), поэтому сайт
работает и в корне домена, и в подпапке проекта (`user.github.io/<repo>/`) — без
настройки `base`. Сборщик не используется; если захотите перейти на Vite, задайте
`base: '/<имя-репозитория>/'`.

## Поведение и взаимодействие

**Мобильные (гироскоп, DeviceOrientation API)**
- `gamma` (наклон влево‑вправо) → горизонтальное смещение, `beta` (вперёд‑назад) → вертикальное.
- Первое показание берётся как **нейтральная точка** (калибровка) — эффект
  относителен тому, как пользователь держит телефон.
- Сырые значения сглаживаются **low‑pass/lerp** фильтром и ограничиваются
  пределами (`±22°`) — без резких скачков.
- **iOS:** feature‑detect `DeviceOrientationEvent.requestPermission`. Если есть —
  на первом экране показывается кнопка **«Нажмите, чтобы начать»**, а
  `requestPermission()` вызывается **внутри обработчика клика**. Подписка на
  `deviceorientation` — только после `granted`.

**Ориентация (только портрет)**
- Интерфейс рассчитан строго на вертикальную ориентацию: он **не растягивается и
  не поворачивается** в ландшафте.
- При повороте телефона в горизонт показывается аккуратная заслонка «Поверните
  телефон вертикально», а вёрстка под ней остаётся ровно портретной; рендер на это
  время ставится на паузу (батарея). Десктоп (точный указатель) не блокируется.
- Это **не мешает** наклону: углы для гироскопа малы (±20°) и не переключают экран
  в ландшафт, так что 3D продолжает реагировать на наклон в портрете.

**Десктоп / fallback**
- Non‑touch устройство → сразу параллакс по `mousemove`.
- Если события ориентации не приходят за **~1 сек** → переключение на тот же
  конвейер трансформаций по `mousemove`.
- Если в доступе отказано, API нет или WebGL недоступен → показывается
  **статичная собранная сцена** (idle‑анимация живёт всегда, пустого фона не бывает).

**Производительность и устойчивость**
- `devicePixelRatio` ограничен `Math.min(window.devicePixelRatio, 2)`.
- Рендер только в `requestAnimationFrame`; сенсор лишь обновляет цель.
- На `visibilitychange` (вкладка скрыта) рендер и слушатели **останавливаются**,
  при возврате — возобновляются без рывка анимации.
- Потеря WebGL‑контекста обрабатывается (`webglcontextlost/restored`): вместо
  «застывшего» канваса показывается фон со статичной подложкой, после
  восстановления цикл возобновляется.
- `prefers-reduced-motion: reduce` → один статичный кадр, без автозапуска анимаций.
- Состояние загрузки (спиннер) показывается, пока 3D не отрисует первый кадр.
- **3D‑движок без внешних зависимостей:** three.js вендорится локально
  (минифицирован, отдаётся с gzip на GitHub Pages); сцена полностью процедурная.
  Единственная внешняя загрузка — брендовые шрифты интерфейсного текста
  (Geologica/Inter с Google Fonts) с системным фолбэком, так что без сети текст
  просто отрисуется системным шрифтом, а сцена не пострадает.

## Деплой на GitHub Pages

**Вариант A — из ветки (проще всего):**
1. Создайте репозиторий и запушьте содержимое этой папки в ветку `main`.
2. Settings → Pages → Build and deployment → Source: **Deploy from a branch** →
   Branch: `main`, папка `/ (root)` → Save.
3. Через минуту сайт будет на `https://<user>.github.io/<repo>/`.

**Вариант B — через GitHub Actions:**
В Settings → Pages выберите Source: **GitHub Actions**. Включённый workflow
`.github/workflows/deploy.yml` опубликует корень репозитория при каждом пуше.

## Локальный запуск

ES‑модули требуют `http://` (не `file://`). Любой статический сервер:

```bash
python -m http.server 8000
# затем откройте http://localhost:8000
```

На десктопе двигайте мышь; на телефоне (через HTTPS, напр. GitHub Pages)
наклоняйте устройство — на iOS сначала нажмите «Нажмите, чтобы начать».
